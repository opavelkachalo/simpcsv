# simpcsv

Простой csv ридер. Читает заданный csv файл и выводит его содержимое в
stdout. Если csv содержит заголовок и индексы, а также формулы типа
`= ARG1 OP ARG2` (без пробелов, они здесь для наглядности), то программа
вычислит их.

Чтобы скомпилировать программу, достаточно перейти в директорию проекта и
дать команду `make`. Запустить программу можно, передав ей аргументом
csv-файл, следующим образом:

```
./simpcsv file.csv
```

Если формат файла некорректный (например состоит из одного столбца, или
индексы не являются положительными числами, или на какой-либо из строчек
число ячеек не совпадает с числом колонок в заголовке), то программа просто
напечатает содержимое входного файла и завершится с ненулевым кодом
возврата.

Если формат файла оказался корректным, но программа не может вычислить
значение ячейки (деление на 0 или зацикливание), то она оставляет
содержимое ячейки без изменения, пытается вычислить значения в других
ячейках и завершается с ненулевым кодом возврата (даже если другие ячейки
вычислить получилось).

Если же и формат файла оказался корректным, и не было ячеек с неправильными
формулами, то программа завершается с кодом 0.

Дополнительно можно вывести таблицу в красивом виде, передав третьим
аргументом флаг `-p` (например, `./simpcsv file.csv -p`). Но этот флаг
работает только если формат файла оказался корректным (см. выше).

# Тесты

В директории `tests/` находится тестовый скрипт (`test.sh`), тестовые csv
файлы в `tests/in/` и файлы с ожидаемым выводом в `tests/out/`. Если
тестовый файл называется `file.csv`, то файл с ожидаемым выводом называется
`file.csv.out`.

Чтобы выполнить тесты, просто запустите `test.sh`. Например, из директории
проекта: `./tests/test.sh`. Также можно передать флаги, чтобы выводились
успешные или неуспешные тесты. Например `./tests/test.sh s` выведет только
успешные тесты, `./tests/test.sh sf` также выведет и проваленные тесты. По
умолчанию используется флаг `f`, т.е. выводятся только проваленные тесты.
